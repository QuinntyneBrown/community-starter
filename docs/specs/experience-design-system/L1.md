# UXDS — Experience and design system

## Product context

The starter must support a recognizable, accessible community experience across anonymous and authenticated surfaces without letting individual features invent competing visual rules. The primary community journey is the proving ground for a small canonical design system, reusable Angular contracts, resilient interaction states, and evidence-backed visual change.

## L1-UXDS-001: Establish one canonical cross-surface visual language

The member application and static public site must consume a single platform-neutral source of foundations, semantic tokens, assets, and documented component states.
