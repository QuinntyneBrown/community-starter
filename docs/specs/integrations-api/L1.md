# INTG — APIs and integrations

## Product context

First-party Angular code, approved external clients, Community automation, and provider callbacks need
stable contracts without bypassing Community scope, Membership policy, privacy, safety, or
Entitlements. Integrations are explicit actors with least-privilege credentials, observable delivery,
bounded retries, and revocable ownership.

## L1-INTG-001: Public API contracts are explicit and safe to evolve

HTTP APIs use stable resource and action semantics, typed boundaries, deterministic pagination,
structured failures, and compatibility rules suitable for generated first-party clients and approved
external consumers.

Community-owned Webhook Endpoints receive only allowed post-commit events through signed,
idempotent, inspectable Deliveries that tolerate outages without duplicating product effects.

## L1-INTG-004: Inbound integrations cannot bypass product policy

Provider callbacks, imports, and connected services are authenticated, bounded, validated, and
translated through ordinary Application use cases so external input cannot create privileged or
