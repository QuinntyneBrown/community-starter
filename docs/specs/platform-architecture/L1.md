# ARCH — Platform architecture

## Product context

The starter is a production-scale, multi-Community platform rather than a compact CRUD tool. It must provide explicit full-stack boundaries, server-owned Community rules, safe relational persistence, and an evolution path that remains legible as Membership, moderation, content, Notifications, and integrations grow. The architecture must make one complete Community journey runnable from a clean checkout without introducing speculative services or hollow layers.

## L1-ARCH-001: Preserve explicit full-stack boundaries

The starter must provide a mandatory layered .NET backend, a library-oriented Angular workspace, a static public site, and one shared design language with dependencies directed toward product policy.
