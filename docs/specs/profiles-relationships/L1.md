# PROF — Profiles and relationships

## Product context

Each Account has one canonical Profile for human-facing identity. Block and Mute provide distinct
safety and attention controls. Community context may add Membership information,
but it never creates a second identity model or bypasses server-owned visibility and Community-isolation rules.

## L1-PROF-001: Present a trustworthy Profile

The platform must provision, update, render, and protect Profile fields and imagery with explicit
ownership, validation, visibility, and lifecycle behavior.
