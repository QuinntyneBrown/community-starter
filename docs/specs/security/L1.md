# SECU — Security and privacy baseline

## Product context

The starter will hold identities and data across multiple isolated Communities, including Memberships, content, moderation records, invitations, uploads, and activity data. Its baseline must prevent client-side trust, cross-Community access, secret leakage, unsafe public input, and insecure delivery shortcuts while making unresolved production risk explicit.

## L1-SECU-001: Model threats and enforce identity in Community context

Identity, Session, Community isolation, invitation, sharing, upload, callback, and destructive-action risks must be documented, with every protected resource authorized by the server in its current Community context.
